# LunaTest DeFi Dashboard Example

`examples/defi-dashboard` is a runnable Vite + React app that dogfoods the built-in LunaTest protocol presets through public package APIs.

It is intentionally deterministic:

- no browser wallet extension is required
- no RPC endpoint is required
- no forked chain is required
- all protocol calls go through the injected `window.ethereum` provider

The app exercises LunaTest L3 frontend-flow support for:

| Protocol | Dogfood path |
| -------- | ------------ |
| Uniswap V2 | `getAmountsOut`, ERC-20 `approve`, router swap-style transaction, receipt |
| Uniswap V3 | Quoter V2 `quoteExactInputSingle`, ERC-20 `approve`, `exactInputSingle`, receipt |
| Curve | `get_dy`, ERC-20 `approve`, `exchange`, receipt |
| Aave | `getUserAccountData`, `supply`, receipt |

L3 support means deterministic frontend-flow coverage. It does not mean exact EVM bytecode execution, exact AMM math, forked liquidity, gas simulation, or historical chain state.

## Run Locally

From the repository root:

```bash
pnpm install --frozen-lockfile
pnpm --filter @lunatest/example-defi-dashboard dev
```

The Vite dev server uses port `5175` by default.

## Validate

From the repository root:

```bash
pnpm --filter @lunatest/example-defi-dashboard test
pnpm --filter @lunatest/example-defi-dashboard build
```

The test suite verifies that the dashboard can materialize all built-in protocol presets, install Luna runtime intercept state, call the injected provider, and render protocol evidence.

## Implementation Notes

- Presets are loaded through `@lunatest/core/browser`.
- Runtime intercept is controlled through `@lunatest/runtime-intercept`.
- The example does not import runtime protocol internals.
- The dashboard snapshot is generated by `src/dogfood.ts` and rendered by `src/app.tsx`.

For support-level details, see [`docs/guides/protocol-support.md`](../../docs/guides/protocol-support.md).
